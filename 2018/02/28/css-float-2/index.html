消除CSS浮动带来的影响

发表于 2018-02-28

消除CSS浮动带来的影响

当我们在进行页面布局的过程里，使用了float属性后。由于浮动元素的特性，就会给布局带来影响。这里就简单总结以下，如何消除浮动所带来的影响。

消除兄弟浮动元素的影响

如图，块b的兄弟元素块a是一个左浮动的元素。

要消除这个浮动元素块a带来的影响，可以使用CSS的clear属性。准确来说，把消除这种影响叫做，清除浮动。

clear属性介绍

clear属性的直观作用，可以说是将元素下移直到没有浮动元素。其有以下4个值可设置：

none：默认值，即不清除；
left：清除左浮动（从右往左看，看到的浮动元素）；
right：清除右浮动（从左往右看，看到的浮动元素）；
both：同时清除左右两边的浮动。

消除子浮动元素的影响

当子元素设置了浮动时，其将不再影响父级元素的高度。如图，由于所有子元素浮动，父级（有蓝色边框）的高度为0。

要消除这种影响，简单来说，就是要使浮动元素能够“撑开”它的父级。把消除这种影响叫做，闭合浮动。

闭合浮动的方法

1. 使用伪元素

给父级元素加上伪元素::after，让其起到“撑开”的作用。

相当于在父级元素的末尾放置一个块级元素，并让它清除左右浮动，如此以来，它就会下移到所有浮动元素下方，以此来“撑开”父级。

2. 利用BFC

BFC（Block Formatting Context,块格式化上下文）,是Web页面的可视化CSS渲染的部分，是块级盒布局发生的区域，也是浮动元素与其他元素交互的区域。

简单来说，我们可以将父级元素创建成为BFC，即可以实现让其内的浮动元素将它“撑开”的效果。具体方法有：

  • 浮动元素 (元素的 float 不是 none)
  • 绝对定位元素 (元素的 position 为 absolute 或 fixed)
  • overflow 值不为 visible 的块元素
  • 内联块元素 (元素具有 display: inline-block)

以上的列举只是部分常用，更多的方法和更详尽的描述可以参考 MDN web docs。

补充

在写清除浮动的demo的时候，发现一个平时比较容易忽略的细节。

现有a，b和c三个div,分别设置为左浮动，不浮动，右浮动。预想中的结果应该是：

但实际情况出现了这样的：

c居然跑到外面去了，这就很尴尬了……说好的浮动以后脱离常规流呢？

一开始以为是CSS代码出问题，最后发现关键是在HTML代码里，是一个很容易被忽略的细节，元素的HTML代码顺序。

出错的顺序是：a、b、c。

正确的顺序是：a、c、b。

个人分析：b放在c之前，浏览器先渲染出了b，但由于b不是浮动元素，先占据了父级元素的空间，跟在其后的c并不能被“撑开”父级，所以只能跟在其后面。

所以，运用到浮动的时候，要注意：浮动元素要编写在HTML的最前面。
